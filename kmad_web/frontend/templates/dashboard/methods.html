Disorder prediction

We first try to find your sequence in the D2P2 database of precomputed disorder predictions. If the sequence is not present there, the disorder prediction is performed with 4 different predictors: DISOPRED, SPINE-D, PSIPRED, and PreDisorder.

KMAD
Knowledge Based Multiple Sequence Alignment for Intrinsically Disordered Proteins

The tool incorporates information from databases and predictions about functional sites to perform a Multiple Sequence Alignment (MSA). The information includes phosphorylations predicted by NetPhos, annotated posttranslational modifications (PTMs) from Swissprot, short linear motifs (SLiMs) from ELM, and functional domains predicted by Pfam. The posttranslational modifications used by KMAD are, the most abundant ones, that is:

  • phosphorylations
  • N-linked glycosylations
  • O-linked glycosylations
  • methylations
  • amidations
  • acetylations
Motifs predicted by ELM are filtered by their GO terms. Motif's list of GO terms is extended by each GO term's parents and all of its descendants. If there is no overlap between sequence GO terms and the extended motif GO terms then the motif is discarded.

KMAD algorithm is a progressive iterative alignment, based on the MaxHom algorithm. Pairiwise alignments follow the Gotoh algorithm with a modified scoring function shown in Figure 1.


Perceived reliability (PR)
PTMs
- perceived reliability of a PTM depends on the type of its annotation source. All predicted phosphorylations get a PR value of 0.3. For annotations from Swissprot we determine the PR value based on its ECO code (Evidence Codes Ontology), representing source of the data. The possible values are: 1, 0.9, 0.8, 0.7. The PR values were chosen arbitrarily for the lack of any other method to quantify the annotation's quality. The ECO codes related to each value are as follows:
1.0 - Experimental evidence codes
  • 0000269, EXP - experimental evidence used in manual assertion
  • 0000314, IDA - direct assay evidence used in manual assertion
  • 0000353, IPI - physical interaction evidence used in manual assertion
  • 0000315, IMP - mutant phenotype evidence used in manual assertion
  • 0000316, IGI - genetic interaction evidence used in manual assertion
  • 0000270, IEP - restriction landmark genome scanning evidence used in manual assertion
0.9 - Computational analysis evidence codes
  • 0000250, ISS - sequence similarity evidence used in manual assertion
  • 0000266, ISO - sequence orthology evidence used in manual assertion
  • 0000247, ISA - sequence alignment evidence used in manual assertion
  • 0000255, ISM - match to sequence model evidence used in manual assertion
  • 0000317, IGC - genomic context evidence used in manual assertion
  • 0000318, IBA - biological aspect of ancestor evidence used in manual assertion
  • 0000319, IBD - biological aspect of descendant evidence used in manual assertion
  • 0000320, IKR - phylogenetic determination of loss of key residues evidence used in manual assertion
  • 0000321, IRD - rapid divergence from ancestral sequence evidence used in manual assertion
  • 0000245, RCA - computational combinatorial evidence used in manual assertion
0.8 - Author Statement evidence codes and Curatorial Statement evidence codes
  • 0000304, TAS - traceable author statement used in manual assertion
  • 0000303, NAS - non-traceable author statement used in manual assertion
  • 0000305, IC - curator inference used in manual assertion
  • 0000307, ND - no biological data found used in manual assertion
0.7 - Automatically assigned evidence code
  • 0000501, IEA - evidence used in automatic assertion
motifs
- The motif's PR value is determined based on its 'probability' P assigned by ELM in form of an E-value. Motifs with P > 0.1 are discarded and in case of overlapping motifs only the one with higher P is kept. PR value calculation is based on formula PR = 1 + 1/log10(P). Therefore the limit of PR is 1 as P tends to 0.
domains
- All domains have PR equal to 1. The reason behind this is that their length is enough to compensate for the lack of the PR factor. If the prediction reliability of a certain domain is low, then when aligning two domains of this kind the overall score for the domains will be anyway significantly lower because of the more frequent mismatches.

ωft weights
The default values of the ωft weights are 10, 3, 3 for PTMs, motifs, ad domains, respectively. Small changes of these values do not seem to have a significant effect on the alignments. We advise keeping the scores for domains and motifs lower than for PTMs as they consist of at least several residues - the score will be added for each two matching residues. Setting too high values might result in forcing alignment of subsequences that should not be aligned with each other.
